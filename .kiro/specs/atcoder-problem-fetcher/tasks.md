# Implementation Plan

## Tasks

- [ ] 1. プロジェクト基盤セットアップ
- [ ] 1.1 Haskell プロジェクト初期化
  - Stack または Cabal でプロジェクトを作成
  - 必要な依存パッケージを追加（optparse-applicative, http-conduit, tagsoup, yaml, text, bytestring）
  - ビルド設定を構成（GHC オプション、警告レベル）
  - _Requirements: 1.1, 1.2_

- [ ] 1.2 (P) 共通型と基本エラー型を定義
  - コマンド型（Fetch、Setup）を定義
  - エラー型（NetworkError、ParseError、FileError）を定義
  - 設定型を定義
  - _Requirements: 1.1, 1.2, 6.3, 6.4_

- [ ] 2. CLI 引数パーサー実装
- [ ] 2.1 サブコマンド構造を実装
  - fetch と setup の2つのサブコマンドを定義
  - サブコマンド未指定時に使用方法を表示
  - 不明なサブコマンドに対するエラー処理
  - _Requirements: 1.1, 1.2, 1.3, 1.4_

- [ ] 2.2 fetch コマンドのオプションを実装
  - URL を位置引数として受け取る
  - `-o/--output` オプションで出力先ディレクトリを指定可能に
  - `-f/--force` フラグで強制上書きを有効化
  - URL 未指定時に使用方法を表示
  - _Requirements: 3.1, 3.5, 7.1, 8.4, 9.6_

- [ ] 2.3 setup コマンドのオプションを実装
  - `--show` オプションで現在の設定を表示
  - `--reset` オプションで設定をリセット
  - オプションなしで対話的設定モードを起動
  - _Requirements: 2.1, 2.5, 2.6_

- [ ] 3. 設定管理機能実装
- [ ] 3.1 (P) 設定ファイルの読み書きを実装
  - 設定ファイルのパスを決定（~/.config/atcoder-fetcher/config.yaml）
  - YAML 形式で設定を保存・読み込み
  - ファイルが存在しない場合はデフォルト値を使用
  - _Requirements: 2.4, 7.2, 7.3_

- [ ] 3.2 対話的設定モードを実装
  - デフォルト出力ディレクトリの入力プロンプト
  - テンプレートファイルパスの入力プロンプト
  - 入力値のバリデーションと保存
  - _Requirements: 2.1, 2.2, 2.3_

- [ ] 3.3 設定表示とリセット機能を実装
  - 現在の設定を整形して表示
  - 設定ファイルを削除してリセット
  - _Requirements: 2.5, 2.6_

- [ ] 4. URL 検証機能実装
- [ ] 4.1 (P) AtCoder URL パターンマッチングを実装
  - 問題ページ URL パターンを認識（/contests/{contest}/tasks/{task}）
  - 問題一覧ページ URL パターンを認識（/contests/{contest}/tasks）
  - コンテスト ID と問題 ID を抽出
  - AtCoder 以外の URL を拒否
  - _Requirements: 3.2, 3.3, 3.4_

- [ ] 5. HTTP クライアント実装
- [ ] 5.1 (P) HTML ページ取得機能を実装
  - http-conduit を使用して GET リクエストを実行
  - User-Agent ヘッダーを設定
  - ネットワークエラーをキャッチして適切なエラー型に変換
  - タイムアウト処理を設定
  - _Requirements: 6.1, 6.3_

- [ ] 6. HTML パーサー実装
- [ ] 6.1 (P) 問題一覧ページのパースを実装
  - HTML から問題リンクを抽出
  - 各問題の URL を生成
  - _Requirements: 4.3_

- [ ] 6.2 問題ページのパースを実装
  - 問題タイトルを抽出
  - 問題文本体を抽出（日本語版を優先）
  - 制約条件を抽出
  - 入出力形式を抽出
  - サンプル入出力を抽出
  - パースエラー時に適切なエラーメッセージを生成
  - _Requirements: 6.2, 6.4_

- [ ] 7. ファイル生成機能実装
- [ ] 7.1 ディレクトリ構造の作成を実装
  - 出力ディレクトリの決定ロジック（コマンドオプション → 設定ファイル → カレント）
  - コンテスト一括取得時にコンテスト名ディレクトリを作成
  - problems/ と answers/ サブディレクトリを作成
  - ディレクトリが存在しない場合は自動作成
  - 作成失敗時のエラー処理
  - _Requirements: 4.1, 4.2, 5.2, 7.1, 7.2, 7.3, 7.4, 7.5_

- [ ] 7.2 (P) Markdown ファイル生成を実装
  - 問題データから Markdown 形式のコンテンツを生成
  - タイトル、問題文、制約、入出力形式、サンプルをセクションごとに整形
  - ファイル名を問題 ID に基づいて決定
  - problems/ ディレクトリに保存
  - _Requirements: 4.5, 5.3, 8.1, 8.2, 8.3_

- [ ] 7.3 Haskell ファイル生成を実装
  - テンプレートファイルの読み込み（設定 → プロジェクト内 → 組み込み）
  - 基本的な main 関数と入出力ボイラープレートを含むデフォルトテンプレート
  - ファイル名を問題 ID に基づいて決定
  - answers/ ディレクトリに保存
  - _Requirements: 4.6, 5.4, 9.1, 9.2, 9.3, 9.4, 9.5_

- [ ] 7.4 ファイル上書き確認を実装
  - 既存ファイルの存在チェック
  - 対話的な上書き確認プロンプト
  - `-f` フラグによる強制上書き
  - _Requirements: 8.4, 9.6_

- [ ] 8. メインフロー統合
- [ ] 8.1 単一問題取得フローを実装
  - URL 検証 → HTML 取得 → パース → ファイル生成の一連の処理
  - 進捗状況の表示
  - エラー発生時の適切な報告
  - _Requirements: 3.1, 3.2, 5.1, 10.1, 10.2, 10.3_

- [ ] 8.2 コンテスト一括取得フローを実装
  - 問題一覧ページから全問題 URL を取得
  - 各問題を順次処理し、進捗を表示（[1/6] 形式）
  - 部分的失敗時も成功分を保持
  - 最終的なサマリーを表示（成功数/総数）
  - _Requirements: 3.3, 4.1, 4.2, 4.3, 4.4, 4.5, 4.6, 4.7, 4.8, 10.1, 10.2, 10.3, 10.4_

- [ ] 8.3 メインエントリポイントを実装
  - CLI パーサーを呼び出してコマンドを取得
  - コマンドに応じて fetch または setup を実行
  - 終了コードの適切な設定
  - _Requirements: 1.1, 1.2_

- [ ] 9. テストとバリデーション
- [ ] 9.1 URL 検証のユニットテストを作成
  - 有効な問題 URL のテスト
  - 有効なコンテスト一覧 URL のテスト
  - 無効な URL のテスト
  - _Requirements: 3.2, 3.3, 3.4_

- [ ] 9.2 HTML パーサーのユニットテストを作成
  - サンプル HTML を使用したパーステスト
  - 問題データの抽出検証
  - パースエラーのハンドリングテスト
  - _Requirements: 6.2, 6.4_

- [ ] 9.3 E2E テストを作成
  - 実際の AtCoder ページを使用した統合テスト（ネットワークモック可）
  - 単一問題取得の完全フロー
  - コンテスト一括取得の完全フロー
  - _Requirements: 4.1, 4.2, 4.3, 4.4, 4.5, 4.6, 4.7, 4.8, 5.1, 5.2, 5.3, 5.4_
